What could we precalculate?* **Option 1: Store sums of all possible subarrays?** This is quickly ruled out as it would require O(n^2) space and preprocessing time, and might not be efficient for lookups.* **Option 2: Store cumulative sums (Prefix sums)?** This looks promising. Let's explore this.**5. Prefix Sums Concept:**The prefix sum at index `i` is the sum of all elements from index 0 up to index `i`. If we have prefix sums, can we use them to calculate the sum of a subarray from `left` to `right`?* **Example:** `nums = [-2, 0, 3, -5, 2, -1]` * Prefix sums: `[-2, -2, 1, -4, -2, -3]` (Let's call this `prefixSums`) * `prefixSums[0] = nums[0] = -2` * `prefixSums[1] = nums[0] + nums[1] = -2 + 0 = -2` * `prefixSums[2] = nums[0] + nums[1] + nums[2] = -2 + 0 + 3 = 1` * ...and so on.* **How to calculate subarray sum using prefix sums?** Let's say we want `sumRange(2, 5)`, which is `nums[2] + nums[3] + nums[4] + nums[5] = 3 + (-5) + 2 + (-1) = -1`. * `prefixSums[5]` (sum up to index 5) = `-3` (which is `-2 + 0 + 3 + -5 + 2 + -1`) * `prefixSums[1]` (sum up to index 1) = `-2` (which is `-2 + 0`) If we subtract `prefixSums[1]` from `prefixSums[5]`, we get `-3 - (-2) = -1`. This is the correct answer! In general: `sumRange(left, right) = prefixSums[right] - prefixSums[left - 1]` **Important Edge Case:** What if `left` is 0? `prefixSums[0 - 1]` doesn't exist. In this case, the sum from index 0 to `right` is simply `prefixSums[right]`. So we can handle this with a conditional or by padding the prefix sum array. Padding with a 0 at the beginning of the prefix sum array will make the formula work even when `left` is 0. * Modified prefix sums (padded with 0): `[0, -2, -2, 1, -4, -2, -3]` * `prefixSums[0] = 0` * `prefixSums[1] = nums[0] = -2` * `prefixSums[2] = nums[0] + nums[1] = -2 + 0 = -2` * ... and so on Now, `sumRange(left, right) = prefixSums[right + 1] - prefixSums[left]` (adjusting indices due to padding). Or, we can just say if left == 0 then return `prefixSums[right]` (using original 0-indexed prefix sum array). Let's stick with padding for simplicity in the formula. So `sumRange(left, right) = prefixSums[right+1] - prefixSums[left]`.**6. Implementing Prefix Sums:*** **`NumArray` Constructor:** * Create a prefix sum array of size `nums.length + 1`. * Initialize `prefixSums[0] = 0`. * Iterate through `nums` from index 0 to `nums.length - 1`. * Calculate `prefixSums[i + 1] = prefixSums[i] + nums[i]`.* **`sumRange` Function:** * Return `prefixSums[right + 1] - prefixSums[left]`.**7. Analyzing Prefix Sum Approach:*** **Time Complexity for `NumArray` initialization:** O(n) to calculate prefix sums.* **Time Complexity for `sumRange`:** O(1) - Constant time lookup and subtraction.* **Space Complexity:** O(n) to store the prefix sum array.This is much better than the naive approach for multiple queries! The initialization takes linear time, but each query takes constant time. This is a classic time-space trade-off.**8.
